# Запустить на маке

Ветка `port/macos`. Собранного `.app` нет и пока не будет: под мак надо
собирать НА маке, а упаковщик (`pyside6-deploy` через Nuitka) ни разу не
проверялся. Запуск из исходников проще и честнее - он покажет то же самое, но
без часа возни с подписью.

## Что понадобится

- macOS 12 или новее
- Python 3.11-3.13 (`brew install python@3.13`)
- Node 20+ (`brew install node`) - для страницы настроек
- `ffmpeg` не нужен

## Команды

```bash
git clone https://github.com/romankandeevy/flowlocal.git
cd flowlocal
git checkout port/macos

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Страница настроек. Без неё окно откроется пустым.
cd web && npm ci && npm run build && cd ..

python app.py
```

## Что скорее всего произойдёт

Честно, по порядку убывания уверенности.

**Импорт пройдёт.** Это единственное, что проверено: `tools/probe_mac_import.py`
подделывает macOS и гоняет все 23 модуля. Прошли все 23. Но проба - не запуск.

**Дальше начнётся Windows.** Всё это Win32 и на маке отвалится:

| Что | Где живёт | Что будет |
|---|---|---|
| Один экземпляр программы | `CreateMutexW` | падение или молчаливый пропуск |
| Окна с сообщением | `MessageBoxW` | нет |
| Глобальный хоткей | `keyboard`, хуки | **не работает - то есть диктовать нечем** |
| Вставка в чужое окно | `win32clipboard`, `SendInput` | нет |
| Фильтр мыши у пилюли | `WM_NCHITTEST` | нет |
| Пауза музыки, слух за сном | `media.py`, `wakeup.py` | нет, они под гардом |

То есть **диктовка на маке пока не работает вовсе**. Смысл запуска сейчас
другой: посмотреть, поднимается ли окно и как выглядит новая страница
настроек.

## Что посмотреть, раз уж запустил

Страница настроек - главное, что тут ново, и она платформенно-нейтральна.
Чтобы открыть её на маке:

```bash
# в config.json рядом с app.py
"ui.web": true
```

`QtWebView` на маке берёт WKWebView вместо WebView2. По документации Qt это
поддерживается; **проверено не было**.

Если окно вообще не поднимется, посмотреть страницу можно и без Qt - она
обычная веб-страница:

```bash
cd web && npm run dev
# открыть http://localhost:5173
```

Так видно витрину контролов в обеих темах. Это не окно программы, но краски,
отступы и шрифты там настоящие.
