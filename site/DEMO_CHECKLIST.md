# Демо-ролик: чеклист записи и конвертации

## ОBS: настройки сцены

- Масштаб Windows: 100% (не 125%, не 150%)
- Запись: 2560×1600 (2× от показа 1280×800)
- FPS: 60
- Формат: MP4, H.264, CFR, CQP 18
- Аудио: 48 kHz, стерео
- Захват: Display Capture (не Window Capture — чтобы показать весь рабочий стол)

## Сценарий посекундно

```
0:00 — пустой рабочий стол, открыто письмо/документ
0:02 — появляется пилюля (живая волна — произносится фраза)
0:03—0:05 — распознавание, текст вставляется в активное окно
0:06 — пилюля гаснет, результат на экране
0:08—0:12 — пустой экран (кадр для петли)
```

## Фраза для диктовки

> встреча в пятницу, нет, в субботу, в десять ноль‑ноль

Намеренно содержит поправку на ходу и «10:00» — демонстрирует две ключевые
возможности без слов.

## Оверлей нажатых клавиш

Не нужен. Зритель видит пилюлю и вставляющийся текст — это нагляднее клавиш.

## ffmpeg: двухпроходный VP9

### Проход 1 (анализ)

```bash
ffmpeg -i demo_source.mp4 \
  -c:v libvpx-vp9 -b:v 0 -crf 32 \
  -deadline best -pass 1 -an -f null NUL
```

### Проход 2 (кодирование)

```bash
ffmpeg -i demo_source.mp4 \
  -c:v libvpx-vp9 -b:v 0 -crf 32 \
  -deadline best -pass 2 \
  -pix_fmt yuv420p -movflags +faststart \
  -vf "scale=1280:800:flags=lanczos" \
  -an demo.webm
```

### H.264 fallback

```bash
ffmpeg -i demo_source.mp4 \
  -c:v libx264 -preset veryslow -crf 22 \
  -pix_fmt yuv420p -movflags +faststart \
  -vf "scale=1280:800:flags=lanczos" \
  -an demo.mp4
```

### Постер (первый кадр, WebP)

```bash
ffmpeg -i demo_source.mp4 -vframes 1 \
  -vf "scale=1280:800:flags=lanczos" \
  poster.webp
```

## Целевые метрики

- Длительность: 8–12 с
- WebM: ≤ 1.5 МБ
- MP4: ≤ 2 МБ
- Постер: ≤ 100 КБ
